# 前端清晰度调优设计

## 目标

在不改变现有页面结构、信息层级和交互布局的前提下，消除当前前端页面的“蒙纱感”“发灰感”和“发虚感”，让导航、卡片和正文层级更清晰。

## 不做的内容

- 不调整页面布局、组件结构或导航信息架构
- 不改动业务逻辑、路由或接口
- 不把当前浅色克制风格改成新的设计语言

## 根因判断

当前视觉发虚主要来自三类样式叠加：

- 顶部导航使用了半透明白底加 `backdrop-filter: blur(...)`，形成持续性的毛玻璃感
- 多组内容卡片使用半透明背景，导致页面表面层缺少足够扎实的承载面
- 次级文字、搜索壳层和阴影对比偏弱，导致内容边界发散

## 方案

采用“清晰化调参”方案，仅修改全局样式 token 与公共表面样式：

- 导航栏改为更实的背景，移除毛玻璃模糊
- 搜索框和关键表面增加更明确的边框与聚焦态
- 常用卡片表面从半透明背景改为实体浅色面
- 阴影收短并减轻扩散，改为更贴近页面的接触阴影
- 保持现有的浅色、克制、编辑器式观感，不改变 UI 编排

## 验证

- 增加一个 CSS 回归测试，约束导航不再使用 `backdrop-filter`
- 约束核心卡片组不再使用半透明 `color-mix(..., transparent)` 背景
- 运行前端测试、lint、build，保证调参不破坏现有行为
